# ХОДЫ

## ХОД: парсинг товаров

### Когда использовать
Когда нужно быстро найти конкретные товары по точным параметрам (модель/объём RAM/SSD/размер и т.д.) на заданных маркетплейсах и вернуть только прямые карточки.

### Рабочий промпт (шаблон)
```text
ХОД: парсинг товаров

Задача: найти товар(ы) по точному совпадению характеристик.

Входные данные:
- Список товаров: <вставить список>
- Сайты/продавцы: <вставить список площадок>
- Для Alibaba использовать только Russian Alibaba (обязательно): https://russian.alibaba.com/ (Alibaba.com не использовать для колонки Alibaba).
- Гео-ограничение поиска: исключать только площадки Украины.
- Поиск в подсанкционных странах разрешён.
- Расширенный набор площадок (приоритет после основных): 1688, Taobao, Made-in-China, GlobalSources, DirectIndustry, IndiaMART, Satu.kz, Deal.by.
- Площадки Украины не использовать (исключены из хода).
- Обязательные параметры совпадения: <например: 16", M1 Pro, 32GB RAM, 512GB SSD>
- Формат результата: ссылка на карточку + цена + валюта

Правила выполнения:
1) Искать строго по обязательным параметрам.
2) Давать только прямые ссылки на карточки товара (не категории, не поиск, не главную).
2.1) Ссылка обязана вести на конкретную карточку именно того товара, который указан в таблице; ссылки на списки похожих/связанных товаров запрещены.
3) Проверять наличие: учитывать только товары, которые прямо сейчас в наличии/доступны к заказу.
4) Если точного совпадения нет или товар не в наличии — в ссылке писать: «Товар отсутствует», а в ячейке цены ставить: "-".
5) Ничего не придумывать: ни ссылок, ни цен.
6) Для каждого найденного товара указывать цену ровно как на сайте + валюту.
6.1) Если карточка есть, но цену извлечь не удалось — в цене ставить "?".
6.5) Российских магазинов искать 4 (а не 2), с приоритетом минимальной цены.
7) Если есть несколько карточек с точным совпадением и в наличии — выбрать наиболее релевантную (по полному совпадению названия/характеристик).
8) Анти-фрод проверка карточки обязательна: если это явно другой товар/чужой бренд/подмена карточки, считать позицию несоответствующей и ставить «Товар отсутствует» (без ссылки и цены). Жёсткую сверку артикулов между площадками не требовать.

Формат ответа:
- Товар: <название>
  - <Площадка 1>: <прямая ссылка или "Товар отсутствует"> | <цена/валюта или "Товар отсутствует">
  - <Площадка 2>: <...>
  - ...
```

### Режим по умолчанию
Строгий режим (приоритетный): только проверяемые и надежные карточки. При сомнении/блокировке/несоответствии — «Товар отсутствует».

### Короткая команда для запуска хода
```text
Применить ХОД: парсинг товаров
```

## ХОД: парсинг товаров (усиленный режим)

### Когда использовать
Когда в обычном режиме слишком мало найденных карточек, особенно для сложных техничных позиций.

### Правила усиленного режима
1) Мультизапрос по каждому товару: минимум 4 варианта запроса
   - точный артикул
   - артикул без разделителей
   - бренд + артикул
   - артикул + ключевые слова товара (ru/en)
2) Двухэтапный отбор:
   - A: собрать расширенный пул кандидатов (10–20)
   - B: отфильтровать на точное совпадение карточки
3) Для Alibaba использовать только Russian Alibaba: https://russian.alibaba.com/
4) Ссылка только на конкретную карточку именно нужного товара (не листинг, не поиск).
5) Анти-фрод обязателен: при несовпадении бренда/модели/фото — отклонять.
6) Статусы:
   - В ссылке: «Товар отсутствует» (если карточки нет)
   - В цене: «-» (если карточки нет)
   - В цене: "?" (карточка есть, но цену достать не удалось)
7) Гео-ограничение: исключать только украинские площадки.
8) Российских магазинов: искать и заполнять 4 магазина (а не 2), с приоритетом минимальной цены.

### Короткая команда для запуска
```text
Применить ХОД: парсинг товаров (усиленный режим)
```
